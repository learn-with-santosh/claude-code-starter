# Angular 19 Project — Claude Memory

## Stack
- **Framework:** Angular 19 (standalone components)
- **Language:** TypeScript (strict mode)
- **State:** Signals + RxJS
- **Styling:** SCSS
- **Mobile:** Capacitor (Android)
- **Desktop:** Electron
- **Package Manager:** npm

## Project Structure
```
src/
  app/
    core/          # Singleton services, guards, interceptors
    shared/        # Shared components, pipes, directives
    features/      # Feature modules (lazy-loaded)
    layout/        # App shell, sidebar, header
  assets/
  environments/
```

## Conventions

### Components
- All components are **standalone** (`standalone: true`)
- Use **signals** for local state (`signal()`, `computed()`, `effect()`)
- Use `inject()` instead of constructor injection
- Template: use `@if`, `@for`, `@switch` (not `*ngIf`, `*ngFor`)

### Services
- One responsibility per service
- Use `providedIn: 'root'` for singletons
- HTTP calls go in dedicated API services only

### Naming
- Components: `feature-name.component.ts` → `FeatureNameComponent`
- Services: `feature-name.service.ts` → `FeatureNameService`
- Files: kebab-case always

### Sidebar
- `app-sidebar` must always be wrapped:
  ```html
  <div class="app-sidebar"><app-sidebar /></div>
  ```
- The sidebar component does NOT render its own outer wrapper div

## Commands
```bash
npm start           # Dev server
npm run build       # Production build
npm run test        # Unit tests (Jest)
npm run lint        # ESLint
npx cap sync        # Sync Capacitor
npx cap run android # Run on Android
```

## Do Not Touch
- `dist/` — build output
- `android/` — generated by Capacitor, do not manually edit Java files
- `*.env` — never modify, never expose

## API Integration
- Base URL from `environment.ts`
- All HTTP calls use typed interfaces
- Use `HttpClient` with `observe: 'response'` for full response access
- Interceptors handle auth headers and error toasts globally
